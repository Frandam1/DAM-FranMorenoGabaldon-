Resumen APP

Parte Estetica/HTML+BOOT

Resumen de los archivos:
  1. urls.py:
    • Importa las funciones y clases necesarias de Django.
    • Define una lista de patrones de URL que vinculan las URL a las vistas correspondientes.
    • Hay cuatro rutas definidas: 'admin/', 'about', '', y 'password-generado'.
  2. navbar.html:
    • Es una plantilla HTML que representa la barra de navegación de la aplicación.
    • Incluye un enlace a la página principal ('/') y a la página "About" ('/about').
  3. home.html:
    • Es una plantilla HTML que representa la página principal del generador de contraseñas.
    • Extiende la plantilla base 'layout.html'.
    • Contiene un formulario donde se puede seleccionar la longitud y las opciones de generación de contraseñas.
    • Al enviar el formulario, se redirige a la vista 'password' para generar una contraseña.
  4. generado.html:
    • Es una plantilla HTML que representa la página de generación de contraseñas.
    • Extiende la plantilla base 'layout.html'.
    • Muestra la contraseña generada en un formato estilizado.
  5. layout.html:
    • Es una plantilla base que define la estructura común de las páginas.
    • Incluye la barra de navegación.
    • Define un bloque de contenido ('content') que puede ser sobrescrito por otras plantillas.

Recuerda que para ejecutar el servidor en Django, debes utilizar el comando python manage.py runserver.

Parte Funcional/DJANGO

El generador de contraseñas implementado en estas funciones de Django tiene las siguientes características:
  • La función home(request) renderiza la plantilla 'home.html', que muestra el formulario para generar una contraseña.
  • La función about(request) renderiza la plantilla 'about.html', que muestra información sobre la aplicación o el generador de contraseñas.
  • La función password(request) se encarga de generar la contraseña según los parámetros proporcionados por el usuario a través del formulario.
  • Se inicializa una lista de caracteres con el alfabeto en minúsculas.
  • La variable generated_password se inicializa como una cadena vacía donde se almacenará la contraseña generada.
  • Se obtiene la longitud de la contraseña deseada del parámetro GET 'lenght' enviado por el formulario.
  • Dependiendo de las opciones seleccionadas por el usuario en el formulario (mayúsculas, caracteres especiales y números), se añaden los caracteres correspondientes a la lista de caracteres.
  • Se utiliza un bucle for para generar una contraseña de la longitud especificada. En cada iteración, se elige aleatoriamente un carácter de la lista de caracteres y se añade a la contraseña generada.
  • Finalmente, se renderiza la plantilla 'password.html', pasando la contraseña generada como contexto para que se muestre al usuario.
